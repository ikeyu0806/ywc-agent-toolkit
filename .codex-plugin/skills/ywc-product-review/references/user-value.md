# User Value Checklist

Analyze whether the service delivers clear, meaningful value to its users.

## Core Questions

1. **Job-to-be-Done clarity**
   - What primary job is the user hiring this service to do?
   - Is that job reflected in the main features and UI flows?
   - Are secondary jobs stealing focus from the primary one?

2. **Value proposition**
   - Is the service's unique value stated or implied clearly?
   - Would a new user understand the benefit within the first interaction?
   - Does the implementation match the stated value proposition?

3. **Feature–value alignment**
   - Do existing features map to clear user goals?
   - Are there features that appear unused or disconnected from user needs?
   - Is there feature bloat that dilutes the core value?

4. **Unmet needs**
   - What adjacent user needs are not yet addressed?
   - Are there common use cases that require workarounds?
   - Is there a gap between what the service offers and what users likely need next?

## Priority Signals

Raise to **High** if:
- The primary JTBD is unclear or inconsistently served across features
- Core features don't map to observable user goals
- A clear unmet need is visible (e.g., manual workarounds, TODO comments on core flows)

Raise to **Medium** if:
- Value proposition is technically correct but poorly communicated
- Feature set is complete but lacks depth in one key area

Raise to **Low** if:
- Minor feature gaps exist alongside a solid core offering
