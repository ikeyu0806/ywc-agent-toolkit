# Starter CLAUDE.md — template with placeholders

This reference is the canonical starter for `ywc-onboard-repo` Phase 4 Output B. Replace every `<...>` placeholder from the Phase 1-3 outputs. Drop sections that are not detectable from the repo rather than guessing; an honest gap is better than a wrong claim.

## Template

```markdown
# Project Instructions

> Generated by ywc-onboard-repo on <YYYY-MM-DD>. Manual edits are preserved on subsequent runs — only the `## Detected Conventions (<date>)` section is rewritten.

## Tech Stack

- Language: <language> <version>
- Framework: <framework> <version>
- Database: <database> (<ORM>)
- Testing: <test-framework>
- Build: <build-tool>
- Lint / format: <linter> / <formatter>

## Code Conventions

- File naming: <kebab-case | camelCase | PascalCase | snake_case>
- Error handling: <throw | Result | error-code | raise>
- Async pattern: <async-await | promise | callback | goroutine | coroutine>
- Test placement: <collocated `*.test.*` | `tests/` directory>
- Test naming: <pattern, e.g., `<feature>.test.ts`>

## Commands

| Task | Command |
|------|---------|
| Dev server | `<command>` |
| Build | `<command>` |
| Tests | `<command>` |
| Lint | `<command>` |
| Format | `<command>` |
| Database (if applicable) | `<command>` |

## Project Structure

| Path | Purpose |
|------|---------|
| `<path>` | <one-line purpose> |
| `<path>` | <one-line purpose> |

## Git Workflow

- Branch prefix: `<feature/ | fix/ | chore/>` (detected from `git branch -a`)
- Commit message: <conventional-commits | freeform> (detected from `git log -n 50`)
- PR workflow: <squash | merge | rebase> (detected from `.github/CODEOWNERS` / repo settings if available)

> If git history is shallow (`< 10` commits), this section should report "Git history insufficient to detect conventions" instead of guessing.

## Detection Confidence

- Detected with high confidence: <N> facts
- Inferred (medium confidence): <N> facts
- Unknown / insufficient signal: <N> facts (listed as `Unknown — <reason>` above)

---

(Below this line, the agent appends per-run findings under a dated header, preserving all content above.)
```

## Rules for filling the template

1. **Every placeholder must be replaced or removed.** A literal `<framework>` in the output CLAUDE.md is a failure — either fill it from detection or delete the row.
2. **No row may claim "all of the above".** "Async pattern: async-await + promise + callback" is a smell — pick the dominant style or report `mixed (no dominant style)`.
3. **Commands come from detected scripts, not assumed defaults.** If `package.json` has no `lint` script, the Lint row is `(none — repo has no lint script)`, not `npm run lint`.
4. **Project Structure rows must trace to Phase 2's Key Directories.** Do not pad with directories the agent did not visit.
5. **Git Workflow section is omitted entirely** when git history is shallow — replaced by a one-line caveat.
6. **The "Detection Confidence" section is mandatory** even when all three counts are non-zero. A missing confidence section hides the fact that some claims are inferred rather than detected.

## When CLAUDE.md already exists

When merging into an existing CLAUDE.md (the common case), use this shape:

```markdown
<...existing CLAUDE.md content preserved verbatim...>

---

## Detected Conventions (<YYYY-MM-DD>)

> Generated by ywc-onboard-repo. Manual edits above this line are preserved.

<the same sections as the template above — Tech Stack, Code Conventions, Commands, Project Structure, Git Workflow, Detection Confidence>
```

The `---` separator and the dated header are mandatory — they signal to the next ywc-onboard-repo run that this section is regenerable, while the content above the separator is not.

## Anti-patterns to avoid

- **Adding the agent's own opinions as conventions.** "Should use named exports" is a recommendation, not a detected convention. Recommendations belong in a separate doc (`docs/CONTRIBUTING.md`), not CLAUDE.md.
- **Inflating the file past 100 lines.** CLAUDE.md is read into every Claude Code conversation — every line costs tokens at every turn. Hard cap at 100 lines; overflow goes into a linked doc.
- **Restating framework documentation.** "Next.js App Router uses file-system routing under `app/`" belongs in the Next.js docs, not CLAUDE.md. CLAUDE.md captures *this repo's* choices.
- **Documenting un-verified workflows.** "Pre-commit hook runs Prettier" is only a fact if `.husky/pre-commit` or `lefthook.yml` exists. Otherwise it is folklore.
