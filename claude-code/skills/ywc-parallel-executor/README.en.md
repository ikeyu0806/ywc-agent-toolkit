# ywc-parallel-executor (Parallel Executor)

A Skill that executes Tasks generated by task-generator in parallel using Agents. It analyzes dependency-graph.md to perform Wave-based parallel execution with Git Worktree isolation.

## Test-first, Deep Module, Critical Module Review

The worker payload carries an interface-first directive and a test-first-where-feasible directive: a behavior-changing task writes a failing test before implementation (docs/config/mechanical are the exception), and the public surface is designed before the body (deep module). When a task's Ownership matches a critical path (auth, payment, crypto, PII, external input), `/ywc-impl-review` and `/ywc-security-audit` are forced at 4d even without `--review`. See `../references/tdd-deep-module-gray-box.md` for details.

## Usage

```text
/ywc-parallel-executor 000001-010-db-create-events           # Single Task
/ywc-parallel-executor 000001-010..000002-040                     # Range (parallel)
/ywc-parallel-executor --all                              # Execute all
/ywc-parallel-executor 000001-010..000002-040 --review            # Parallel + auto Review
/ywc-parallel-executor 000001-010..000002-040 --local-merge       # Local merge without PR
/ywc-parallel-executor 000001-010..000002-040 --draft             # Create Draft PR (human merges later)
/ywc-parallel-executor 000001-010..000002-040 --per-task-pr       # Per Task: create PR, CI, review, merge
/ywc-parallel-executor 000026-010..000026-030 --aggregate-pr --group-name payments --pr-lang ko  # One PR per group
```

## Options

| Option | Description |
|--------|-------------|
| `--tasks-dir <path>` | Tasks directory path (default: tasks/) |
| `--pr-lang <lang>` | Language for the final PR (`--aggregate-pr`) title/description (default: auto-detect) |
| `--review` | Auto-run /ywc-impl-review after each Task (combinable) |
| `--local-merge` | No PR, merge and push to base-branch directly |
| `--draft` | Create a single Draft PR after all Tasks complete (left open for a human to merge) |
| `--per-task-pr` | Per Task: create PR → wait for CI → handle bot reviews → merge (full lifecycle, like sequential-executor's default) |
| `--aggregate-pr` | Whole invocation → one branch + one PR. Tasks still run in parallel (Waves) and accumulate onto a single aggregate branch, then the end-of-run PR is marked ready, CI-verified, bot-reviewed, and **merged** (the full-lifecycle twin of `--draft`) |
| `--group-name <name>` | Names the aggregate branch (`aggregate/<name>`) and disambiguates concurrent groups. `--aggregate-pr` only; defaults to `aggregate/<base-branch>-<timestamp>` |

> There is no default mode. The user must explicitly choose one of `--local-merge` / `--draft` / `--per-task-pr` / `--aggregate-pr`. The four are mutually exclusive.

## Group Execution (`--aggregate-pr`)

Use `--aggregate-pr` when many Tasks should be **grouped and delivered as one PR per group**, optionally with **groups running in parallel**.

**Core idea**: one invocation = one group = one PR. Tasks **within** a group run in parallel by Wave, and each Task's changes plus its `chore: mark ... as completed` marker commit accumulate onto a single aggregate branch. After all Waves finish, that branch opens one PR and runs ready → CI → bot review → merge-readiness gate → merge → base sync in a single pass. Because the marker commits already live on the branch, there is no separate Mark Complete after the merge.

### Single Group

```text
/ywc-parallel-executor 000026-010..000026-030 --aggregate-pr --group-name payments --review --pr-lang ko
```

- `--review`: gate each Task's worktree branch through `/ywc-impl-review` before it joins the aggregate
- `--pr-lang ko`: fix the PR title/description language
- Preview the Wave plan (task order, dependencies, mode) with `--dry-run` before committing to the run.

### Parallel Groups (concurrent)

To run several groups at once, isolate each with **one clone per group** (independent `.git`). `--aggregate-pr` accumulates directly onto the local `<base>` branch (`git reset --hard origin/<base>` + per-task merges), so two groups touching the same `<base>` overwrite each other's accumulation. **Git worktrees are not enough**: they separate the working tree and the untracked `.ywc-run-state.json`, but share `.git` refs — the local `<base>` branch is shared (and checkout-exclusive), so the collision persists at the ref layer.

```bash
git clone <repo-url> ../grp-payments && \
  ( cd ../grp-payments && /ywc-parallel-executor 000026-010..000026-030 \
      --aggregate-pr --group-name payments --review --pr-lang ko ) &

git clone <repo-url> ../grp-search && \
  ( cd ../grp-search && /ywc-parallel-executor 000027-010..000027-040 \
      --aggregate-pr --group-name search --review --pr-lang ko ) &

wait
```

To stay in one repo with no extra clones, run groups **back to back** (each group is still internally parallel):

```bash
/ywc-parallel-executor 000026-010..000026-030 --aggregate-pr --group-name payments --pr-lang ko
/ywc-parallel-executor 000027-010..000027-040 --aggregate-pr --group-name search   --pr-lang ko
```

See [references/aggregate-pr.md](./references/aggregate-pr.md) for the full procedure and concurrency-safety rules.

## Execution Flow

1. Parse dependency-graph.md
2. Plan Waves (Topological Sort)
3. Execute per Wave: Create Worktree → Parallel Agent execution → Verify (Task Verify + full/impacted regression suite + Ownership scope gate) → Merge → Delete Worktree

## Task → Agent Auto-Mapping

| Category | Agent |
|----------|-------|
| db, api, domain, lib, worker | Backend Agent (sonnet) |
| ui | Frontend Agent (sonnet) |
| test | QA Agent (sonnet) |
| infra | DevOps Agent (sonnet) |
| refactor | Reviewer Agent (opus) |

Override with Agent Hint:
```markdown
## Parallel Execution Metadata
- Agent Hint: frontend
```

## Comparison with sequential-executor

| Scenario | Recommended Tool |
|----------|-----------------|
| Small scope (1-3 Tasks) | sequential-executor |
| Strong sequential dependencies | sequential-executor |
| Large scope (4+ Tasks) | /ywc-parallel-executor |
| Many parallelizable Tasks | /ywc-parallel-executor |
