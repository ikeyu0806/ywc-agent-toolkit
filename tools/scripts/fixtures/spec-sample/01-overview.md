# Project Overview

This sample project is a **booking management** service for boutique studios.
It exists to *replace* spreadsheet-based scheduling with a shared, mobile-first
flow. The product targets studio owners with **fewer than 50 rooms**.

## Goals

- Reduce double-booking incidents to zero within the first quarter.
- Provide same-day visibility into capacity for staff and customers.
- Stay operable by a single non-technical administrator.

## Out of Scope

The first release intentionally excludes:

1. Payment processing (handled by an upstream POS).
2. Multi-currency pricing.
3. Native mobile apps — the web app is responsive enough for launch.

> "We do not need a perfect product. We need a product that prevents the
> next double-booking phone call." — operations lead
